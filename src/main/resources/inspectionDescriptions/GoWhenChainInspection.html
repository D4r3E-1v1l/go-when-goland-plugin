Reports structural and semantic issues in go-when matcher chains.

This inspection checks that matcher chains are explicit, complete, and safe to maintain. Every completed matcher chain must end with a terminal method.

Valid terminal methods are:

  • Else(...)
  • ElseDo(...)
  • Exhaustive()

Use Else(...) or ElseDo(...) when a fallback branch is expected. Use Exhaustive() when all valid cases are expected to be covered.

The inspection reports errors for:

  • matcher roots without completed branches or terminal methods
  • conditions such as Case(...), Range(...), When(...), or Pattern(...) without Then(...) or ThenDo(...)
  • invalid conditions, actions, or terminals for the current matcher type
  • duplicate terminal methods
  • terminal methods that are not the last call in the chain
  • numeric condition overlap
  • unreachable numeric conditions

Numeric condition checks use strict interval semantics. Numeric Case(...) values are treated as closed point intervals. For example, Case(3) is treated as [3, 3].

Range helpers are interpreted as:

  • when.Range(a, b): [a, b)
  • when.Closed(a, b): [a, b]
  • when.From(a): [a, +∞)
  • when.Until(b): (-∞, b)

The inspection reports a warning when Exhaustive() is used with a supported enum type but some enum cases are not covered.

The current enum exhaustive check supports enums declared as type Name int or type Name string in the current file.

The quick fix can add .Exhaustive() after the last completed branch when a terminal method is missing.
